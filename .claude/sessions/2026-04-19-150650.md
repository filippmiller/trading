## Technical Decisions

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Grid sweep in-memory vs N SQL queries | 400 rows × 30 cols ≈ 100KB hydrated once; each combo = pure arithmetic ~30ms | Separate query per combo (slow, DB-dependent) |
| `startDay` param threaded into `evaluateExitWalk` | Supports `entryDelayDays` without duplicating walk logic | Create a separate "delayed walk" function (duplication) |
| 10,000-combo hard cap | Prevents 8-axis × 5-value explosion (390k combos) from DoS-ing the server | Soft limit + warning (too easy to ignore) |
| Breakeven arm as first-class exit | Common real-world stop not expressible with hard_stop+trail alone | Compose via trail with 0 activation (fragile semantics) |
| Ship Grid Sweep on separate branch from PR #8 | Independent shipping — tab audit isn't blocking Grid Sweep | Stack on PR #8 (would block Grid Sweep on PR #8 review) |
| Keep everything as-is for deployment | User's call after seeing tradeoffs | Cloudflare Tunnel + Pages (complex, no auth work done) |

## Gotchas & Notes for Future Agents

- **SSH tunnel is fragile.** `scripts/tunnel-db.sh` uses `ssh -f -N` which daemonizes but doesn't auto-reconnect on drop. Until a watchdog exists, sporadic API 500s ("Failed to fetch entries." / `ECONNREFUSED`) usually = tunnel dead. First debug step: `netstat -ano | grep ':3319'`. Restart: `ssh -f -N -o ExitOnForwardFailure=yes -o ServerAliveInterval=30 -L 3319:127.0.0.1:3320 root@89.167.42.128`.
- **Silent catches pattern is systemic.** After fixing `/settings`, `/strategies`, found same on `/paper`, `/runs`, `/voice`, `/markets`, `/signals`. Pattern: `.catch(() => setSymbols([]))` or `catch {}` silently swallowing network errors. Future fix = one-shot audit + ESLint rule.
- **Grid Sweep expects `d1_morning IS NOT NULL`** as gate. Entries without intraday bars are skipped (ACTIVE cohorts that just enrolled will be filtered out until next sync). This is intentional — simulating without intraday data would be noise.
- **`reversal_entries.consecutive_days` = 1 for all 400 COMPLETED rows** (legacy, column added later). Only ACTIVE rows have real streak data. Any strategy research that filters on `consecutive_days >= N` will only hit ACTIVE rows with partial windows until more data accumulates.
- **Two PRs open simultaneously, both unmerged.** PR #8 touches 18 files across UI, PR #9 builds on scenario-simulator. If PR #9 merges first, PR #8 will need rebase. If PR #8 merges first, PR #9 picks up the hydration/trust fixes via merge. **Recommend merging #8 first** because it's pure cleanup and unblocks everything else.
- **Auth gap blocks any public deployment.** User declined auth work for now but this is the next real blocker before Cloudflare/Vercel/anything.
- **`final_pnl_usd` at $100 position size numerically equals `final_pnl_pct`** by design (both computed from directed % change, position size baked in). This is coincidental simplification — if position size ever becomes non-$100, SQL CASE in `syncActiveSurveillance` needs updating.
