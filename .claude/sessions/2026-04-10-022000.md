### Phase 11: Idempotent Enrollment Fix (66 Tickers Bug)
- User spotted April 8 cohort had 66 tickers instead of 20
- Investigation via SQL showed multiple enrollment runs across the day + container restarts:
  - 09:33 Apr 8: original 20
  - 13:45 Apr 8: 19 more (different Yahoo top 10)
  - 01:49-02:19 Apr 9: 26 more (container restarts)
- Root cause: `jobEnrollMovers()` and `autoEnrollTrenders()` re-fetched Yahoo movers on every run. Yahoo's top 10 changes throughout the day → new symbols inserted. `ON DUPLICATE KEY UPDATE` only dedupes same-symbol-same-date.
- **Fix**: Both functions now check `SELECT COUNT(*) FROM reversal_entries WHERE cohort_date = ?` BEFORE fetching movers. If count > 0, log "SKIP: already enrolled N tickers" and return.

## Technical Decisions

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| VPS MySQL as single source of truth | Cron must be always-on; web app is local-only dev | Deploy web app to VPS (complexity) |
| SSH tunnel for local dev | Zero config change to .env.local (3319 → 3320) | Expose MySQL publicly (security risk) |
| Trading-day loop over calendar-day loop | Calendar loop wrote wrong columns for Friday cohorts | None — critical bug |
| COALESCE in backfill | Prevents overwriting live intraday with daily approximations | Always overwrite (loses precision) |
| Yahoo `range=60d` as primary source | 60 trading days in one call, zero cost, beats every paid tier | Twelve Data primary (800/day limit) |
| Symbol-level cache per sync | ~30x reduction in API calls (30 per symbol → 1) | Per-call fetches (hit rate limits) |
| Filter bars to time windows at parse | 256M container OOM'd without filtering | Store everything, bigger container |
| Twelve Data circuit breaker check BODY not HTTP | TD returns 200 OK with `code: 429` in body | Check HTTP status only (would miss quota) |
| Container memory 256M → 1G + NODE_OPTIONS | 60-day fetches need heap headroom | Smaller per-symbol window (defeats purpose) |
| Build paper trading in-app vs external API | Alpaca + Tradier both block Canada (verified via Playwright) | IBKR (full KYC + fund), Moomoo (mobile app), Questrade (retail API blocked) |
| $100k default paper account | Matches Alpaca default, room for multiple positions | Lower amount limits strategy testing |
| MARKET orders fill synchronously in POST | User expects immediate feedback | Deferred async fill (worse UX) |
| LIMIT/STOP orders fill on GET | Simple, no cron needed | Dedicated cron (overkill) |
| Idempotent enrollment via COUNT(*) check | Simplest possible check, zero state | Dedicated "enrolled_dates" table (overkill) |
| Keep first 20 entries for April 8, delete rest | 09:33 sync is the "morning opening" snapshot the strategy targets | Pick larger/later sync (less representative) |

## Gotchas & Notes for Future Agents

1. **VPS MySQL is the single source of truth.** Web app connects via `bash scripts/tunnel-db.sh` (localhost:3319 → VPS:3320).
2. **SSH tunnels on Windows are flaky.** Ports get stuck in TIME_WAIT. Use different local ports if 3319 is stuck.
3. **Market holidays are hardcoded** in both `surveillance.ts` and `surveillance-cron.ts` (2026 + 2027). Update annually.
4. **SYNC_SECRET is optional** — auth only enforced if env var is set.
5. **Canada is NOT accessible on Alpaca paper-only** — verified 2026-04-09 via Playwright. Alpaca docs lie about "anyone globally". Signup form literally has no Canada option.
6. **Tradier is also blocked from Canada** — forces US brokerage KYC after signup. Don't waste time on it.
7. **Paper trading lives in-app.** Do NOT try external paper trading APIs. Extend `src/lib/paper.ts` and `/api/paper/*`.
8. **Enrollment is idempotent per cohort_date.** Log line `SKIP: already enrolled N tickers` is correct behavior. To force re-enrollment, DELETE existing rows for that cohort_date first.
9. **Yahoo's unadvertised 60-day 5-min endpoint**: `?interval=5m&range=60d` returns ~4700 bars per symbol — this is the whole foundation of the cron now.
10. **Twelve Data circuit breaker** prevents 258×1-3s = 8min waste when quota exhausted. Resets at start of each sync. TD returns body `{code: 429}` with HTTP 200 (NOT HTTP 429) — check body not status.
11. **Container memory is 1G now, not 256M.** Don't shrink without testing — OOM on 60-day fetches.
12. **NODE_OPTIONS=--max-old-space-size=768** is required. Don't remove.
13. **tsconfig excludes `scripts/surveillance-cron.ts`** because it uses `node-cron` from the standalone Docker package.json (not main). Don't re-include.
14. **Backfill script uses COALESCE** — safe to re-run without overwriting live intraday data.
15. **API keys in `.env.local` and VPS .env**: TWELVEDATA_API_KEY works for historical intraday. FINNHUB_API_KEY and FMP_API_KEY are real-time quote only (historical blocked on free tier) — not used by cron but saved for future use.
16. **The "Live Surveillance" tab at /reversal shows 0 active tickers** — pre-existing UI bug that filters out entries without intraday prices. Matrix and Historical Audit tabs work correctly.
17. **Order matching engine runs on every GET /api/paper.** No dedicated cron. Pending limit/stop orders fill on next page load. For production auto-matching, add a cron call to `fillPendingOrders()`.
18. **`paper_trades.quantity` is computed at fill time** from `investment_usd / fill_price`. Old trades may have quantity=0 — API falls back to `investment_usd / buy_price` in that case.
19. **`paper_equity_snapshots` table exists but isn't populated yet.** Reserved for equity curve chart.
20. **MAAS is the 1 missing cell** in April 8 cohort — Yahoo 5-min chart data gap, not a code bug. 59/60 cells is 98.3% fill rate.
21. **The main trading algorithm is NOT yet implemented.** Analysis from April 7 showed only "BUY >7% loser, hold 3d" was profitable (62% win rate, +1.9% avg over 3 months). Implementing this as an automated paper trader is the next logical feature.
