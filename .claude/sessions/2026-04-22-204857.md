## Technical Decisions

| Decision | Rationale | Alternatives Considered |
|---|---|---|
| Batch endpoint bypasses RTH + live-price | User's mental model is "fill at yesterday's matrix close" — requires arbitrary caller-supplied price. Paper only, safe tradeoff. Flagged via `is_manual_fill=1` | Special order_type, override flag on single-order route — both hackier |
| Partial-success semantics on batch | Real retail brokers handle multi-leg this way; user submitting 3 tickers doesn't want 1 failure aborting the other 2 | All-or-nothing transaction (deferred as optional toggle) |
| Lazy whitelist sync + in-migration backfill | MVP curated-CSV seed was explicitly partial per doc comment; Yahoo Movers only returns real US equities so lazy-insert is safe by construction | Manual `--refresh` (ops toil), UI-only signal without fixing source |
| Phase 2 (stop-eval) = reuse existing cron | `jobMonitorPaperTrades` already does it perfectly every 15 min RTH with PR #33 slippage parity | New EOD cron (duplication) |
| Price-deviation band = 20%, fails open on missing reference | Penny/low-float names can open 10-15% off prior close, 20% catches "typo" territory but allows volatile realism. Fails open on data gap so fresh-symbol flow isn't broken | Close-fail (too restrictive), daily OHLC clamp (heavier), no band (Codex interview hole) |
| `is_manual_fill` boolean, not `execution_mode` enum | Single bit is enough for analytics-filter purpose. Enum can be added later without schema breakage | Enum `MANUAL_PRICE`/`LIVE_QUOTE` (scope creep for MVP) |
| `exchange='LAZY_SYNC'` marker vs NULL | Makes lazy-added rows distinguishable from "we don't know the exchange"; enables future enricher script targeting just lazy rows | NULL (foot-gun for future exchange-filters), dedicated provenance column (schema bloat) |
| One-shot scripts deleted after use | User preference: Playwright smoke/E2E/cleanup scripts are session-scoped, don't belong in repo | Commit as reusable tooling (rejected) |
| Grid Sweep drops `breakevenAtPct` silently | Form's `trailingActivateAtPct` is a different mechanic; mapping them would obfuscate which strategy the user is actually testing | Keep old mapping (semantic lie), show a warning modal (scope creep) |
| UI sections show empty-state vs vanishing | Consistency with Open Positions. Vanishing confuses users into thinking the feature doesn't exist | Keep vanishing (status quo); conditionally render filters only |
| Matrix opacity dim only when selection active | Visual de-emphasis requires something to contrast against; empty selection = nothing in scope = don't fade | Always-dim-unchecked (PR #29 behavior, user called out as broken), remove F2 cue entirely |

## Gotchas & Notes for Future Agents

- **Pages that always show empty-state headers:** `/paper` Open Positions + Trade History + Pending Orders. Don't regress by re-introducing `.length > 0 &&` wrappers around the entire section.
- **`paper_orders.quantity` is NULL for all batch-endpoint fills.** Real quantity lives on `paper_trades.quantity`. Idempotent replay SELECTs MUST `LEFT JOIN paper_trades` — asymmetric handling = dormant bug (fixed in #41).
- **`is_manual_fill=1` is the canonical provenance flag for batch fills.** Do NOT infer provenance from `order_type='MARKET'` — batch inserts MARKET orders that weren't against live quotes.
- **`exchange='LAZY_SYNC'` in `tradable_symbols` means "added at enrollment, not curated".** Any future exchange-filter code must accept this value or explicitly decide to exclude lazy rows.
- **Deviation band fails OPEN when `prices_daily` has no reference.** Missing row = no sanity check = fill proceeds. This is intentional for fresh-symbol flow; weigh carefully before changing.
- **Batch endpoint is order-dependent.** Rows process sequentially, each fill mutates account cash before the next. Reordering the same payload produces different outcomes at buying-power edges. Idempotency via `client_request_id` ensures literal retries work, but reorderings don't.
- **Matrix opacity-40 dim is guarded on `selectedRowIds.size > 0`.** Don't revert to unconditional — that's what made the grid look washed-out.
- **GH CLI TLS is unstable on this Windows box.** For merges/comments use curl+token fallback: `curl -sSf -X PUT -H "Authorization: Bearer $(gh auth token)" -H "Accept: application/vnd.github+json" -d '{"merge_method":"squash"}' "https://api.github.com/repos/OWNER/REPO/pulls/NUM/merge"`.
- **User strongly prefers one-shot verification scripts DELETED after use.** Don't commit Playwright smoke/E2E/cleanup scripts even if they seem reusable.
- **`agent-log.md` immutable policy: no rewrites.** Errata go into NEW entries, even for your own mistakes.
- **Zod upper bounds are typo-guards, NOT strategy policy.** E.g. `trailing_stop_pct max=50` rejects fat-finger $500k trails, not legitimate 30% volatile-name strategies. Don't tighten to "strategy-correct" values.
- **`/log` session notes MUST be committed immediately.** Branch-switching without commit drops them to stash and they can be forgotten. Commit to the current branch before `git checkout`.
- **SYSTEM.md (`.claude/SYSTEM.md`) is the single place for system-level context.** Read it before touching paper/matrix/surveillance code. If you learn something new that future agents should know, add it there rather than re-deriving.
