## Context

User asked to verify the trading app's surveillance worker correctly fetches and stores ticker data. This evolved into a full session covering: fixing the data pipeline, building a matrix view, backfilling 3 months of historical data, running statistical strategy analysis, and building a paper trading interface.

## What Was Done

### Phase 1: Fix Surveillance Worker
- Files: `src/lib/data.ts`, `src/app/api/surveillance/sync/route.ts`
- Changes: Stooq API was blocking automated requests. Added Yahoo Finance chart API as fallback in `fetchDailyBars()`. Added missing `ensureSchema()` to sync route. Removed `consecutiveDays >= 2` filter — now enrolls all top 10 gainers + top 10 losers daily.
- Reasoning: Stooq returns error message instead of CSV, silently breaking trend analysis. Yahoo provides identical OHLCV daily data.

### Phase 2: Matrix Tab
- Files: `src/app/reversal/page.tsx`, `src/lib/reversal.ts`, `src/app/api/reversal/route.ts`
- Changes: Promoted "Matrix" to top-level tab. Extended `ReversalEntry` type from d1-d3 to d1-d10 (30 measurement fields). Fixed API route to return all 10 days of data (was hardcoded to d1-d3 only). Added legend bar, actual dates per cohort, tooltips, sorting by trigger magnitude, dollar prices + % in each cell. Full-width layout with sticky scroll.
- Reasoning: Matrix is the core visualization — rows=tickers, columns=M/D/E per day. Shows at a glance whether mean reversion works.

### Phase 3: Historical Backfill (3 months)
- Files: `scripts/backfill-matrix.ts`
- Changes: Script fetches 503 S&P 500 tickers from Wikipedia, downloads daily bars from Yahoo (`range=3mo`), computes top 10 gainers/losers per day, inserts into reversal_entries with 10-day follow-up close prices. Supports `BACKFILL_RANGE` env var.
- Result: 1,200 entries across 60 trading days, 21,800 price points filled.

### Phase 4: Strategy Analysis (SQL)
- No code changes — ran extensive SQL queries against the backfilled data
- Tested: all combinations of BUY/SHORT × losers/gainers × 1d/2d/3d/5d/10d holds × >5%/>7%/>10% triggers
- Also tested: overnight holds, streak-based signals, martingale viability, consecutive down-day distributions
- Key finding: **BUY >7% loser, hold 3 days** — 62% win rate, +1.90% avg return across 224 trades over 3 months. Only consistently profitable strategy.

### Phase 5: Paper Trading
- Files: `src/app/api/paper/route.ts`, `src/app/paper/page.tsx`, `src/components/AppShell.tsx`
- Changes: New `paper_trades` table. API route fetches live prices from Yahoo, supports sell action. Page shows portfolio with live P&L updating every 30 seconds, "ПРОДАТЬ" button to close positions. Added to sidebar navigation with "New" badge.
- 5 initial trades recorded: AXTI ($41.99), PAYP ($18.84), FIGS ($13.40), SEDG ($45.09), SOC ($14.18) — all >7% losers on April 7.

## Technical Decisions

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| Yahoo Finance as Stooq fallback (not replacement) | Preserves existing behavior when Stooq works | Full replacement — but Stooq may work for some users |
| 3-month backfill range | Enough data for statistical significance (~200+ trades per strategy) | 1 month (too few trades), 6 months (Yahoo may not provide) |
| BUY >7% loser, hold 3d as primary strategy | Only strategy with >60% win rate sustained over 3 months | Overnight hold (dropped to 52% on 3mo), short gainers (negative P&L on 3mo) |
| Paper trading in-app vs external platform | User wanted to monitor and sell within the day; external platforms also recommended (Webull, TradingView) | Could have only recommended external platforms |
| Extend ReversalEntry to d1-d10 in type + API | DB already had d4-d10 columns but type and API mapper were stuck at d3 | Could have used dynamic `any` typing — but explicit types catch bugs |

## Problems Encountered

- **Stooq API blocking**: Returns error message instead of CSV. Discovered by testing raw fetch from Node.js vs Next.js. Fixed with Yahoo fallback.

- **ReversalEntry type missing d4-d10**: DB had columns but TypeScript type only defined d1-d3. MatrixCell rendered undefined as empty. Fixed by extending type.

- **Playwright timeouts**: 400-entry page too large for snapshots. Worked around with evaluate() for scrolling.

## Gotchas & Notes for Future Agents

1. **Stooq is unreliable** — always ensure Yahoo Finance fallback. If Yahoo also blocks, consider Polygon.io or Alpha Vantage.
2. **The backfill uses `BACKFILL_RANGE` env var** — default `1mo`, set to `3mo` for full backfill. Yahoo max is typically `6mo` for daily bars.
3. **Paper trades are in MySQL `paper_trades` table** — local Docker container on port 3319. Credentials: root/trading123, db: trading.
4. **Strategy analysis results (3 months, S&P 500)**: BUY >7% loser hold 3d = 62% win, +1.9% avg. All other strategies <55% win rate or negative P&L. Overnight hold dropped from 67% (1mo) to 52% (3mo) — not reliable.
5. **The consecutiveDays filter was intentionally removed** from autoEnrollTrenders. The design is: 10 gainers + 10 losers daily, no filtering. Streak data is metadata only.
6. **Live paper trading prices refresh every 30 seconds** — only works during market hours (9:30 AM - 4:00 PM ET).
7. **Matrix cell shows two lines**: dollar price (top) and direction-adjusted % change (bottom). Tooltip has full details.
8. **User communicates in Russian** — the paper trading UI uses Russian labels (ПРОДАТЬ, Вложено, etc.).
