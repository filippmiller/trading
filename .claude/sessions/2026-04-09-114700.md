### Phase 8: Yahoo 60-Day Rewrite (Hypothesis Verification)
- Re-verified the "free tiers don't include historical intraday" hypothesis

- **Rewrote fetchIntradayPrice** with symbol-level caching:
  - `SymbolBarCache` — per-sync cache keyed by symbol
  - `fetchYahoo60d()` — single fetch per symbol, 60 days of bars
  - `fetchTwelveDataDay()` — fallback, only called if Yahoo returns empty for that symbol
  - `lookupBar()` — instant in-memory lookup from cached bars

## Technical Decisions

| Decision | Rationale | Alternatives Considered |
|----------|-----------|------------------------|
| VPS MySQL as single source of truth | Cron must be always-on; web app is local-only dev | Deploy web app to VPS (unnecessary complexity) |
| SSH tunnel for local dev | Zero config change to .env.local (port 3319 maps to VPS:3320) | Expose MySQL publicly (security risk) |
| Trading-day loop over calendar-day loop | Calendar loop wrote Monday's price to d4 instead of d1 for Friday cohorts | None — critical bug, one correct fix |
| COALESCE in backfill | Prevents overwriting live 5-min intraday data with daily-bar approximations | Always overwrite (loses precision) |
| Market holidays hardcoded set | Simple, no external dependency, update annually | API-based holiday calendar (complexity) |
| Yahoo `range=60d` as primary source | 60 trading days in one call, zero cost | Twelve Data primary (800/day limit) |
| Symbol-level cache per sync | ~30x reduction in API calls (from 30 per symbol to 1) | Per-call fetches (hit rate limits immediately) |
| Filter bars to time windows at parse | Memory pressure with 80 symbols × 4700 bars each | Store everything (OOM at 256M container limit) |
| Twelve Data circuit breaker | Each 429 response takes 1-3s → 258 calls × 2s = 8 min waste | Retry each call (slow) or hardcode disable (inflexible) |
| Container memory 256M→1G + NODE_OPTIONS | 60-day bar fetches exceed default Node heap on tight container | Smaller per-symbol window (defeats purpose) |
| Build paper trading in-app vs external API | Already have infrastructure; no country restrictions; zero rate limits | Alpaca (blocked), Tradier (blocked), IBKR (full KYC + funding) |

## Problems Encountered

- **Two separate databases**: VPS cron had its own MySQL, web app had local MySQL. Fixed by making VPS the single source.
- **SSH tunnel ports stuck in TIME_WAIT**: Port 3319 wouldn't rebind after tunnel drops. Worked around by using 3332 → 3345.
- **Next.js dev server cached stale DB connection pool**: Had to kill+restart with explicit DATABASE_URL after changing tunnel port.
- **Twelve Data quota burned in 10 minutes**: The cron tried to fill 2000+ NULL midday cells (backfill only filled morning/close). Marked old backfilled entries COMPLETED to drop from 226 → 86 active, DLQ cleared.
- **Cron sync hung for 8+ minutes**: Initially suspected Twelve Data fallback loop (each 429 takes 1-3s × 258 cells). Added circuit breaker. Then container OOM'd at 256MB → bumped to 1GB + NODE_OPTIONS heap size.
- **Bar data memory**: 4700 bars × 80 symbols in cache caused OOM. Fixed by filtering to target time windows only at parse time (~600 bars/symbol).
- **Research claimed Alpaca paper-only works from Canada**: Verified via Playwright — FALSE. Canada is not in the country dropdown. Research was outdated (referenced old forum post).
- **Research claimed Tradier sandbox is email-only**: Verified via Playwright — FALSE. Signup redirects to full brokerage KYC with SSN/Tax ID.
- **Finnhub and FMP free tiers advertised historical intraday**: Both blocked in practice, endpoints return "access denied" or "restricted endpoint" on free plans. Free tier tiers changed in 2024-2025.

## Gotchas & Notes for Future Agents

1. **VPS MySQL is the single source of truth.** Web app connects via `bash scripts/tunnel-db.sh` (localhost:3319 → VPS:3320).
2. **SSH tunnels on Windows are flaky.** Ports get stuck in TIME_WAIT. Use different local ports if 3319 is stuck.
3. **Market holidays are hardcoded** in both `surveillance.ts` and `surveillance-cron.ts`. Update annually (currently has 2026 + 2027).
4. **SYNC_SECRET is optional** — auth only enforced if env var is set.
5. **The Live Surveillance tab shows 0 tickers** even with entries — pre-existing UI filter issue, not related to this session. Matrix and Historical Audit tabs work.
6. **Backfill script uses COALESCE** — safe to re-run without overwriting live intraday.
7. **Yahoo's 60-day endpoint** (`?interval=5m&range=60d`) is the key discovery — 60 trading days of 5-min bars in one call, unadvertised by Yahoo.
8. **Twelve Data circuit breaker** prevents 258×1-3s = 8min waste when quota exhausted. Resets at start of each sync.
9. **Container memory is 1G now, not 256M.** If you shrink it back, expect OOMs on the 60-day fetches.
10. **NODE_OPTIONS=--max-old-space-size=768** gives Node extra heap headroom. Don't remove it without testing.
11. **API keys in `.env.local`**: TWELVEDATA_API_KEY is working; FINNHUB_API_KEY and FMP_API_KEY are for real-time quotes only (historical intraday blocked on free tiers).
12. **Alpaca is NOT accessible from Canada** — confirmed. Ignore any research claiming otherwise. The signup form literally doesn't have Canada in the country dropdown.
13. **Tradier is also blocked** from Canada — forces US brokerage KYC after basic signup.
14. **Paper trading should be built into the app**, not integrated with external APIs. The `paper_trades` table and `/api/paper/route.ts` already exist. Next session: extend these.
15. **Twelve Data quota resets at UTC midnight**, not any other timezone. 800 credits/day, 8/min rate limit.
16. **Yahoo daily bar counts**: AAPL returns 4681 5-min bars for 60 days — that's ~80 bars/day × 60 days. Stored in memory during parse.
